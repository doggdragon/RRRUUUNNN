【RA4M2_MINI_DEMO 传送带控制代码答辩说明】

文档目的：
- 让你在答辩时能讲清楚：代码在做什么、为什么这么做、改哪个参数会有什么实际效果。
- 对应主文件：RA4M2_MINI_DEMO/src/hal_entry.c

==================================================
1. 项目整体功能（一句话版本）
==================================================
该程序基于瑞萨 RA4M2，实现“传送带电机速度控制 + 外部门控启停 + 编码器测速反馈 + OLED显示”。

核心思路：
- 用编码器测速度；
- 用“前馈 + PI”算法算出PWM占空比；
- 用100us周期的软件PWM驱动TB6612的PWMA；
- 用P001作为外部运行门控，低电平强制停机，高电平允许运行；
- OLED开机显示“CONVEYOR BELT”。

==================================================
2. 硬件引脚与实际功能
==================================================
- TB6612_PIN_PWMA = P104：电机PWM输出（软件控制占空比）。
- MOTOR_RUN_GATE_PIN = P001：外部运行门控输入（带上拉）。
  - 高电平/悬空：允许运行
  - 低电平：停止运行
- ENC_PIN_A = P000，ENC_PIN_B = P200：编码器AB相输入（测速）。
- OLED_PIN_SCL = P408，OLED_PIN_SDA = P407：OLED软件I2C。

注意：当前代码中“电机方向”是硬件固定，不在软件里切换方向。

==================================================
3. 主循环是怎么跑的（时序）
==================================================
- MAIN_TICK_US = 100us：主循环每100us执行一次。
- 每次循环都执行 motor_apply_output()，输出软件PWM。
- 每10个100us（即1ms）执行：
  1) conveyor_auto_profile_update()：处理门控输入（去抖）
  2) encoder_poll_1ms()：读取编码器AB相并累计计数
- 每20ms（CONTROL_PERIOD_MS）执行 conveyor_control_update()：
  - 计算当前速度
  - 计算目标斜坡
  - 执行PI控制
  - 更新PWM占空比

可讲点：
“这是典型的分层循环调度：快环负责PWM输出，慢环负责控制算法，避免控制计算过于频繁导致噪声和算力浪费。”

==================================================
4. 关键函数逐个讲（答辩重点）
==================================================
4.1 app_peripheral_init()
- 作用：统一配置IO方向、上拉、初始电平。
- 意义：确保上电后引脚状态可控，避免误动作。

4.2 motor_apply_output(uint16_t pwm_permil)
- 输入：千分比占空比（0~1000）。
- 方法：内部把0~1000映射到0~100，再用100步相位计数输出高低电平，形成软件PWM。
- 结果：P104产生等效PWM驱动信号。

4.3 encoder_poll_1ms()
- 作用：每1ms读取编码器AB相。
- 方法：利用四相译码表（quad_table）判断增量方向，更新 g_encoder_count。
- 意义：获得速度反馈，供闭环控制使用。

4.4 conveyor_auto_profile_update()
- 作用：处理外部门控输入（P001）并去抖。
- 去抖策略：高低电平都做20ms确认（GATE_STOP_DEBOUNCE_MS）。
- 结果：更新 g_conveyor.run_cmd（系统该不该运行）。

4.5 conveyor_control_update()（核心）
分5步：
1) 速度测量：根据20ms窗口内编码器计数算 motor_rps，再换算 belt_speed_meas_mps。
2) 参考速度斜坡：让目标速度逐步变化，避免机械冲击。
3) 误差计算：err = 参考速度 - 实测速度。
4) 控制律：
   duty = 前馈 + Kp*err + Ki*积分(err)
5) 限幅与保护：
   - 占空比限制在 [0, PWM_MAX_PERMIL]
   - 起转最小占空比 PWM_MIN_RUN_PERMIL
   - 积分限幅防止积分饱和失控

可讲点：
“前馈决定大致扭矩，PI负责纠偏；这样比纯PI响应更快、更稳。”

4.6 conveyor_display_init() / conveyor_display_title_once()
- 初始化OLED并显示开机标题 “CONVEYOR BELT”。

==================================================
5. 速度换算原理（答辩常问）
==================================================
- 先由编码器脉冲得到电机转速（rps）。
- 再根据齿轮/轮径关系换算到皮带线速度。

关键参数：
- ENCODER_CPR = 52（每圈计数）
- MOTOR_GEAR_DIAMETER_MM = 12.0
- BELT_PULLEY_DIAMETER_MM = 24.9

如果这些机械参数填错，会导致“显示速度”和“真实速度”不一致。

7. “改哪些值会发生什么变化”（最重要）
==================================================
A. 目标速度类
1) TARGET_BELT_SPEED_MMPS
- 改大：传送带目标速度变快。
- 改小：目标速度变慢。
- 风险：太大可能接近PWM上限，出现跟不上/震荡。

B. 启停平顺性
2) MAX_ACCEL_MPS2
- 改大：加减速更快，更灵敏但冲击更大。
- 改小：更平滑，但启动到目标速度更慢。

C. PI控制强度
3) PI_CTRL_KP
- 改大：响应更快，但过大可能抖动。
- 改小：更稳但响应慢。

4) PI_CTRL_KI
- 改大：稳态误差消除更快，但过大易低频振荡/超调。
- 改小：稳态误差可能残留。

D. 输出限制
5) PWM_MAX_PERMIL
- 改大：允许更高驱动能力、最高速度可能更高。
- 改小：限扭保护更强，但最高速度下降。

6) PWM_MIN_RUN_PERMIL
- 改大：低速更容易“转起来”，但最低可控速度上升。
- 改小：低速更细腻，但可能起不来或间歇抖动。

E. 前馈参数（影响基础扭矩）
7) PWM_FF_BASE_PERMIL
- 类似“起步基础油门”。
- 改大：低速更有劲，但空载可能偏快。

8) PWM_FF_GAIN_PER_MPS_PERMIL
- 速度越高，额外给多少占空比。
- 改大：高速区更有力；过大可能超调。

F. 测速与显示平滑
9) SPEED_MEAS_WINDOW_MS
- 改大：测速更稳但延迟更大。
- 改小：响应快但噪声变大。

10) SPEED_DISPLAY_FILTER_ALPHA
- 改大：显示更跟手但更抖。
- 改小：显示更平滑但更滞后。

11) SPEED_ZERO_DEADBAND_MPS
- 改大：低速更容易显示0（抗抖更强）。
- 改小：更灵敏，但零速附近可能跳动。

G. 门控与安全
12) GATE_STOP_DEBOUNCE_MS
- 改大：抗干扰更强，但启停响应变慢。
- 改小：响应更快，但可能误触发。

13) ENABLE_ENCODER_STALL_FALLBACK
- 1：启用编码器失效降级逻辑（容错）。
- 0：关闭降级（当前为0）。

==================================================
8. 建议的调参顺序（实操）
==================================================
建议按顺序，不要乱跳：
1) 先定目标速度：TARGET_BELT_SPEED_MMPS
2) 再定可用占空范围：PWM_MIN_RUN_PERMIL、PWM_MAX_PERMIL
3) 再调平顺性：MAX_ACCEL_MPS2
4) 再调控制器：先Kp后Ki
5) 最后调显示与去抖参数

经验：每次只改1~2个参数，记录“改前/改后”响应曲线。

==================================================
9. 答辩可直接背的“高频问答”
==================================================
Q1：为什么用“前馈+PI”，不用纯PI？
A：纯PI在负载变化时响应慢，前馈提供基础驱动力，PI只做误差修正，响应更快更稳。

Q2：为什么要加速度斜坡？
A：避免目标值突变导致机械冲击、电流尖峰、皮带打滑。

Q3：为什么要做门控去抖？
A：现场传感器/按键容易毛刺，不去抖会出现误启停。

Q4：系统如何保证安全？
A：门控低电平可强制停机；占空比限幅；可选编码器失效降级逻辑。

Q5：你把速度改成8倍改的是哪里？
A：改的是 TARGET_BELT_SPEED_MMPS（5.0 -> 40.0），这是目标线速度参数。
